# H2O Thermo Calc React

React/Vite-версія веб-калькулятора теплофізичних властивостей води, льоду, перегрітої та насиченої водяної пари. Проект призначений для заміни старої статичної реалізації `calc_water` на компонентну React-архітектуру.

Поточна версія: **2.0.0**

## Можливості

- Розрахунок властивостей води за температурою і тиском.
- Автоматичне визначення фази: лід, рідка вода або перегріта пара.
- Окремий режим для властивостей насиченої пари:
  - розрахунок за температурою насичення;
  - розрахунок за тиском насичення;
  - картки результатів у тому ж стилі, що й основні властивості води;
  - окрема історія розрахунків насиченої пари.
- Підтримка одиниць температури `°C`, `K`, `°F`.
- Підтримка одиниць тиску `MPa`, `kPa`, `bar`, `atm`.
- Інтерактивний графік властивостей від температури.
- Історія останніх розрахунків у `localStorage`.
- Перемикання мов: українська, англійська, польська, німецька.
- Світла, темна та автоматична тема.

## Розрахункова модель

Проект використовує інженерні апроксимації для швидкого браузерного розрахунку:

- густина рідкої води: поліном Kell з корекцією за тиском;
- температура насичення і тиск насичення: рівняння Антуана;
- теплота пароутворення: кореляція Watson;
- властивості перегрітої пари: ідеальний газ для густини та інженерні наближення для transport-властивостей;
- температура плавлення льоду Ih: наближення за IAPWS R14-08;
- властивості льоду Ih: наближені кореляції для густини, теплоємності, теплопровідності, ентальпії та швидкості звуку.

Це навчально-інженерний калькулятор. Для критичних промислових розрахунків потрібно звіряти результати з офіційними таблицями або повною реалізацією IAPWS-IF97.

## Структура проекту

- `src/App.jsx` - головна сторінка, вкладки калькулятора, результати та історія.
- `src/components/` - компоненти інтерфейсу: шапка, форма введення, картки властивостей, графік, панель насиченої пари, футер.
- `src/context/` - провайдери теми та локалізації.
- `src/hooks/` - React-хуки для теми, локалізації та історії.
- `src/utils/water.js` - формули та функції розрахунку властивостей.
- `src/utils/i18n.js` - словники перекладів.
- `src/version.js` - версія застосунку, яка відображається у футері.
- `src/App.css` - стилі застосунку.

## Запуск

Встановити залежності:

```bash
npm install
```

Запустити dev-сервер:

```bash
npm run dev
```

Зібрати production-версію:

```bash
npm run build
```

Перевірити ESLint:

```bash
npm run lint
```

Локально застосунок відкривається за адресою, яку покаже Vite, зазвичай `http://localhost:5173/`.

## Заміна старого GitHub/Vercel проекту

Щоб старий QR-код продовжив працювати, потрібно зберегти ту саму публічну адресу, на яку він веде.

Найнадійніший сценарій:

1. Замінити вміст старого GitHub-репозиторію файлами з цього проекту.
2. Залишити той самий Vercel project, який уже підключений до старого репозиторію.
3. У Vercel перевірити build settings:
   - Framework Preset: `Vite`;
   - Build Command: `npm run build`;
   - Output Directory: `dist`;
   - Install Command: `npm install`.
4. Запустити новий deploy.

Для цього проекту QR-код веде на стабільну адресу:

```text
https://calc-water.vercel.app/
```

Після redeploy у тому самому Vercel project ця адреса відкриватиме нову React-версію, тому QR-код міняти не потрібно.

Якщо QR-код веде на конкретний immutable deployment URL Vercel, наприклад URL з унікальним хешем старого deploy, замінити вміст за цією адресою не вийде. У такому разі треба або перевипустити QR-код, або налаштувати користувачів на стабільний домен проекту.

## Перевірено

- `npm.cmd run build` - проходить.
- `npm.cmd run lint` - проходить після очищення старих lint-помилок.
- Вкладка насиченої пари перевірена в браузері: розрахунок за температурою працює, картки властивостей відображаються, історія розрахунків додається та відновлює попередній результат.
