# H₂O Thermo Calc

Веб-калькулятор теплофізичних властивостей води на основі апроксимаційних формул IAPWS-IF97 (рідка фаза, діапазон 0–370 °C, тиск 0.001–100 МПа).

## Структура проєкту

- `index.html` — розмітка сторінки
- `styles.css` — стилі
- `app.js` — UI-логіка калькулятора
- `water.js` — формули властивостей води
- `i18n.js` — переклади та логіка перемикання мов (UA / EN / PL / DE)
- `version.js` — константа `APP_VERSION`

## Запуск

Статичний сайт; обслуговується через `npx http-server` на порту 5000 (workflow `Start application`).

## Журнал змін

### v1.8.0
- Додано графік залежності обраної властивості від температури при поточному тиску.
- Випадаючий список для вибору властивості (ρ, cₚ, λ, μ, ν, a, Pr, β, h).
- Дві окремі гілки на графіку: рідка вода (синя) та перегріта пара (червона), розділені вертикальною лінією температури насичення (помаранчева, з підписом).
- Вертикальна пунктирна лінія позначає поточну температуру розрахунку.
- Автоматична логарифмічна шкала Y, якщо діапазон значень перевищує 50× (наприклад для густини: 1000 → ~0.5 кг/м³).
- Графік реалізовано як SVG з адаптацією до теми через CSS-змінні; легенда та підписи перекладені 4 мовами.
- З'являється автоматично після першого розрахунку.

### v1.7.0
- Додано підтримку трьох тем: світла / темна / авто (за налаштуванням системи).
- Перемикач тем у шапці поряд із вибором мови (іконки Sun / Auto / Moon).
- Вибір зберігається в `localStorage` під ключем `theme`.
- Тема застосовується ще до першого рендеру (вбудований скрипт у `<head>`), щоб уникнути миготіння.
- Реалізовано через CSS-змінні: світла палітра в `[data-theme="light"]`, темна — за замовчуванням; для `auto` через `@media (prefers-color-scheme: light)`.
- Підказки до кнопок перекладено всіма чотирма мовами.

### v1.6.0
- Набір властивостей тепер залежить від фази:
  - **Рідка вода**: ρ, cp, λ, a, μ, ν, Pr, σ, β, Pₛ, h, **r** (теплота пароутворення при T_насич для поточного P).
  - **Перегріта пара**: ρ, **v** (питомий об'єм), cp, **γ** (показник адіабати), λ, a, μ, ν, Pr, β, **c** (швидкість звуку), h, **r** (теплота пароутворення).
- Для пари прибрано поверхневий натяг та тиск насиченої пари — у газовій фазі вони не мають фізичного сенсу.
- Теплота пароутворення обчислюється за кореляцією Watson (h_fg = 262.1·(Tc − T)^0.38), коректна аж до критичної точки 374 °C.
- Швидкість звуку та γ для пари — за моделлю ідеального газу (γ = cp/cv, c = √(γ·R_s·T)).
- Додано переклади нових властивостей для всіх чотирьох мов.

### v1.5.1

- Додано підказку при наведенні (4 мови).

### v1.5.0

- T_насич додано в експорт CSV та PDF.
- Додано переклади мітки T_насич для всіх чотирьох мов (UA / EN / PL / DE).

### v1.4.0
- Додано підтримку перегрітої водяної пари: режим (рідина / пара) визначається автоматично за тиском і температурою насичення.
- Властивості пари: густина — за рівнянням ідеального газу (R_s = 461.5 Дж/(кг·К)); cp, λ, μ — інженерні поліноми/лінійні наближення; ентальпія через теплоту пароутворення.
- Верхню межу температури підвищено з 370 °C до 800 °C.
- Виправлено переклад мітки фази "steam" з "Перегріта рідина" на "Перегріта пара" (та аналогічно для EN/PL/DE).
- Прибрано блокування при T > T_насич — тепер показуються властивості пари замість помилки.
- Поверхневий натяг для газової фази показується як «—» (не визначено).
